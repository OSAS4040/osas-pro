# ماذا تشمل هذه الحزمة؟ وهل يكفي «رفع ملف على السيرفر»؟

## باختصار

| السؤال | الجواب |
|--------|--------|
| هل هذه الحزمة **تطبيقاً جاهزاً** أرفعه على السيرفر فيعمل؟ | **لا.** حجمها صغير لأنها **وثائق + سكربت إرشادي** فقط. |
| هل يكفي **رفع هذه الحزمة فقط** دون Git أو بناء؟ | **لا.** Laravel + Vue يحتاجان على الخادم (أو في CI): **كود المشروع** أو **حزمة إصدار كاملة**، ثم `composer`، و`migrate`، و`npm run build` للواجهة (أو Docker يغطي ذلك). |
| ماذا تفيد الحزمة إذن؟ | توحيد **الفهم** (نطاق، API، مسارات، إعدادات، اختبارات) وتسهيل **تنفيذ الأوامر** عبر السكربت المرفق **بعد** أن يكون المستودع أو نسخة الإصدار موجودة على الخادم. |

---

## ما الذي يُعدّ «رفعاً مباشراً» في مشاريع مثل هذا؟

1. **الطريقة المعتادة (موصى بها):**  
   على السيرفر: `git pull` (أو نشر من Pipeline) على **فرع/وسم الإصدار** → ثم أوامر `composer` + `migrate` + بناء الواجهة.  
   السكربت `deploy_after_git_pull.sh` في هذه الحزمة **يلخّص** هذه الأوامر من جذر المشروع.

2. **حزمة إصدار كاملة (بدون Git على الخادم):**  
   في المستودع يوجد مسار **`deployment/official_release_package/`** — حزمة مشروع جاهزة للنسخ للخادم مع خطوات في `DEPLOYMENT_STEPS.md` **داخل تلك الحزمة**.  
   هذا **ليس** ملف الـ zip الصغير `handover_phone_web_TEAM_PACKAGE.zip`؛ الحزمة الرسمية **ضخمة** وتُبنى/تُحدَّث كجزء من إجراء الإصدار عندكم.

3. **Docker:**  
   إن نشركم عبر `docker compose`، فالرفع «المباشر» غالباً هو تحديث الصور/المجلد ثم `docker compose up` و`migrate` داخل الحاوية — راجع `docker-compose.yml` في جذر المشروع.

---

## ماذا يشمل محتوى الأرشيف `handover_phone_web_TEAM_PACKAGE.zip`؟

- وثائق تسليم (`TEAM_HANDOVER_FULL_AR.md` وملحقات).
- **لا يشمل:** مجلد `vendor`، ولا `node_modules`، ولا `frontend/dist` جاهزاً — لأن ذلك يُنتج على البيئة التي تبني فيها الإصدار.

---

## ماذا يحتاج فريقكم فعلياً للنشر؟

1. الوصول إلى **نفس إصدار الكود** (Git tag/branch أو حزمة official release).
2. ضبط **`backend/.env`** (قاعدة بيانات، Redis، Twilio / SMS، وعدم تفعيل OTP وهمي في الإنتاج).
3. تشغيل **الترحيلات** الخاصة بالجوال (`phone_otps`, `registration_profiles`, تعديلات `users`).
4. **بناء الواجهة** ونشر `dist` خلف nginx كما تعودتم.
5. تشغيل **قائمة التحقق** في `TEAM_HANDOVER_FULL_AR.md`.
