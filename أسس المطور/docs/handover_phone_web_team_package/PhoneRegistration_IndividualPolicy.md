# سياسة المستخدم الفرد بعد التسجيل بالجوال + OTP

## النطاق التقني (محدّث)

مسار **الجوال + OTP** وواجهات إكمال الملف والمراجعة مُنفَّذ **لتطبيق الويب فقط** (Vue 3 + واجهات Laravel). عميل **`mobile_field_client`** (Flutter) خارج نطاق هذا المسار حتى يُقرَّر لاحقاً صراحةً.

## الحالة بعد إدخال الاسم (`individual_completed`)

1. **الهوية**: رقم الجوال مُوثَّق، الاسم الكامل محفوظ على المستخدم وملف `registration_profiles`.
2. **التشغيل**: لا يوجد `company_id` ولا فرع ولا اشتراك مستأجر — **لا لوحة ورشة كاملة** ولا صلاحيات محاسبة/مخزون.
3. **الدور**: يبقى `phone_onboarding` حتى يُنشئ المستخدم نشاطاً تجارياً عبر مسار التسجيل التقليدي (`/register`) أو ينتقل لحساب مرتبط بشركة (دخول بالبريد/كلمة المرور بحساب مدعو).

## مسارات مقصودة (GO)

| المسار | الوصف |
|--------|--------|
| **شاشة إرشاد** | `/phone/onboarding/done` — توضيح الحالة وروابط واضحة. |
| **استكشاف عام** | مسموح بزيارة الصفحات ذات `meta.publicPage` (مثل `/landing`) أثناء الجلسة دون مغادرة مسار الجوال؛ يظهر شريط عودة على الهبوط. |
| **إنشاء شركة لاحقاً** | زر يُسجّل الخروج ثم يوجّه إلى `/register` لإنشاء مستأجر تجريبي/تشغيلي حسب حوكمة المنصة الحالية. |
| **انضمام لشركة** | زر يُسجّل الخروج ثم `/login` لاستخدام دعوة أو بيانات صادرة عن مالك الشركة. |

## ما لا نفعله في هذه المرحلة

- لا نُنشئ شركة تشغيلية أو فرعاً أو محفظة تلقائياً من حساب «فرد» فقط.
- لا نُغيّر الدور إلى `owner` إلا عبر مسارات التسجيل/الدعوة القائمة أو موافقة المنصة حيث ينطبق ذلك.

## مراجعة

عند تغيير سياسة المنتج (مثلاً لوحة محدودة للفرد داخل التطبيق)، حدّث هذه الوثيقة و`PhoneOnboardingDoneView.vue` وPHPDoc في `CompleteRegistrationProfileService`.
