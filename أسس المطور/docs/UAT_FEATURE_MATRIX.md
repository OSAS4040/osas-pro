# UAT feature matrix — موجز للقبول اليدوي

**آخر تحديث:** 2026-04-05

الغرض: قائمة مرجعية سريعة لحالات **Ready / Partial / HIDDEN / DISABLED** وما يجب التحقق منه يدوياً.

| الميزة | الحالة | ملاحظات قبول |
|--------|--------|----------------|
| مستندات المنشأة `/documents/company` | **PARTIAL** | قائمة المستندات (وصفية) عبر `documents_registry` في إعدادات الشركة؛ **محتوى الملفات لا يُرفع للخادم**؛ رسائل تحميل/حفظ/خطأ واضحة؛ ترحيل لمرة واحدة من `localStorage` القديم إن وُجد. |
| الأرشفة الإلكترونية `/electronic-archive` (و`/workshop/hr-archive`) | **READY** | عرض جدول حقيقي من `GET /companies/{id}/settings` → `documents_registry`؛ بحث/فلترة؛ **عرض/تحميل** للمرفق فقط إن وُجد في **نفس الجلسة** (`sessionStorage` بعد إضافة ملف من مستندات المنشأة)؛ وإلا رسالة صريحة بعدم توفر الملف مركزياً. عطّل المسار بـ `VITE_ELECTRONIC_ARCHIVE=false`. |
| تصدير PDF فاتورة / جواز مركبة | **Ready** | راجع `docs/Feature_Readiness_Audit.md` §5. |

## 3) تطبيق شروط الظهور (واجهة الموظف — 2026-04-05)

قرار الإظهار موحّد بين **الراوتر** (`access-denied` مع سبب)، **القائمة الجانبية**، **البحث السريع**، **Command Palette**، **اختصارات اللوحة/التقارير**، و**شريط الأيقونات السفلي** عبر `frontend/src/config/staffFeatureGate.ts`.

| الميزة / المسار | مرئي عندما | مخفي عندما | ممنوع الوصول (مباشر) عندما |
|-----------------|------------|------------|------------------------------|
| ذكاء الأعمال `/business-intelligence` | `VITE_INTELLIGENCE_COMMAND_CENTER` مفعّل **و** بوابة `intelligence` مفتوحة للمستأجر (المالك يتجاوز المصفوفة) | العلم معطّل أو القسم `intelligence` غير مفعّل لغير المالك | نفس الشروط → `access-denied` (`feature`) |
| مركز العمليات الذكي `/internal/intelligence` | ما سبق **+** صلاحية `reports.intelligence.view` | أي شرط أعلاه ناقص | بدون صلاحية → `permission`؛ بدون علم/قسم → `feature` |
| بيارات / حجوزات / اجتماعات `/bays*`, `/bookings*`, `/meetings*` | بوابة `operations` مفتوحة (المالك يتجاوز) | القسم `operations` مغلق لغير المالك | `access-denied` (`feature`) |
| الخريطة الحرارية `/bays/heatmap` | كمسارات التشغيل أعلاه (نفس بوابة `operations`) | كما أعلاه | كما أعلاه |
| هيكل القطاعات `/settings/org-units` | مدير + `org_structure` في ملف النشاط (المالك يتجاوز المصفوفة) + صلاحيات الإعدادات | أي شرط ناقص في الواجهة | `access-denied` (`manager` / `feature` حسب الحارس) |
| مسارات الورشة `/workshop/*` | بوابة `hr` مفتوحة (المالك يتجاوز) | القسم `hr` مغلق لغير المالك | `access-denied` (`feature`) |

## 4) أخطاء الـ API والتحميل (واجهة الموظف — 2026-04-05)

مرجع تفصيلي: **`docs/Frontend_Error_Handling_Standard.md`**.

| ما يُختبر يدوياً | التوقع |
|------------------|--------|
| تسجيل الدخول (بيانات خاطئة / تحقق) | رسالة واضحة في الصفحة **بدون** تكرار Toast من الـ interceptor |
| POS — إتمام البيع | زر معطّل أثناء الطلب؛ رسالة خطأ موحّدة عند الفشل |
| إنشاء فاتورة / أمر شراء / GRN / أمر عمل | تعطيل الإرسال + رسالة خطأ أو Toast واحد معنٍ |
| دفع فاتورة / استرداد / وسائط | نفس النمط؛ لا رسائل خام تقنية |
| إعدادات الشركة (حفظ / رفع) | عند الفشل يظهر Toast من **`apiClient`** (لا ازدواجية مع «فشل الحفظ» محلياً) |
| انقطاع الشبكة / بطء | Toast «اتصال» أو «مهلة» حسب الحالة |
