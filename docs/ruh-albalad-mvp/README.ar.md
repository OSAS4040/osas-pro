# روح البلد - خطة تنفيذ MVP (نسخة تشغيل فعلية)

هذه الحزمة تترجم فكرة المشروع إلى تنفيذ عملي سريع وقابل للاختبار.

## الهدف من هذه النسخة

- تشغيل مشهد افتتاحي بضغطة زر.
- دعم اختيار سيناريوهين: `ازدهار` و`إهمال`.
- ربط الإضاءة الديناميكية مع الصوت.
- إبقاء التعقيد منخفضا لضمان الثبات أثناء العرض.

## نطاق MVP التقني

- متحكم: `ESP32 DevKit V1`.
- إضاءة: شريط `WS2812B` (مثال 60 LED).
- صوت: `DFPlayer Mini` + سماعة 3W.
- مدخلات: زر بدء + زر سيناريو A + زر سيناريو B + حساس قرب اختياري.

## هيكلة الملفات

- `BOM.ar.md`: قائمة القطع المطلوبة.
- `WIRING-ESP32.ar.md`: Pin Map نهائي + Wiring Diagram مبسط.
- `STATE-MACHINE.ar.md`: منطق الحالات والتسلسل التشغيلي.
- `RUNBOOK-MVP.ar.md`: خطوات الرفع، الاختبار، وFail-safe.
- `code/esp32/ruh_albalad_mvp.ino`: كود ESP32 الأساسي.
- `code/esp32/ruh_albalad_mvp_demo.ino`: نسخة اللجنة السريعة (45-60 ثانية).
- `ACCEPTANCE-CHECKLIST-DEMO.ar.md`: قائمة قبول نهائية قبل العرض.
- `RUNBOOK-DEMO-DAY.ar.md`: دليل يوم العرض (تشغيل/فحص/إعادة تشغيل).
- `code/voice_bridge/voice_bridge.py`: جسر صوتي اختياري للأوامر.

## خطوات البدء السريع

1. راجعي قائمة القطع في `BOM.ar.md`.
2. نفذي التوصيل حسب `WIRING-ESP32.ar.md`.
3. ارفعي كود `ruh_albalad_mvp.ino` إلى ESP32.
4. حملي ملفات الصوت إلى microSD داخل DFPlayer:
   - `0001.mp3`: افتتاحي.
   - `0002.mp3`: سيناريو ازدهار.
   - `0003.mp3`: سيناريو إهمال.
   - `0004.mp3`: خاتمة.
5. اختبري التسلسل حسب `STATE-MACHINE.ar.md`.
6. اتبعي `RUNBOOK-MVP.ar.md` لاختبار الدورة الكاملة قبل العرض.

## ملاحظات تنفيذ مهمة

- استخدمي مزود 5V خارجي للإضاءة والصوت، وليس USB فقط.
- جميع الأجهزة يجب أن تشترك في `GND` واحد.
- لا تضيفي أي مكونات جديدة قبل نجاح دورة التشغيل الأساسية كاملة.
- التشغيل الأساسي في MVP لا يعتمد على `voice bridge`.
- نسخة اللجنة `ruh_albalad_mvp_demo.ino` تبقي `voice bridge` خارج النطاق.
