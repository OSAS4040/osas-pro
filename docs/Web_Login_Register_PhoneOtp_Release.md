# نشر الويب — تسجيل الدخول والتسجيل ومسار الجوال + OTP

**الغرض:** ملف واحد يلخّص ما يخص **صفحة تسجيل الدخول** و**صفحة التسجيل** و**مسار الدخول/التسجيل بالجوال (OTP)** لاستخدامه عند النشر أو التسليم للفريق.

**النطاق:** تطبيق الويب فقط (Vue 3 + واجهات Laravel ` /api/v1 `). لا يشمل `mobile_field_client`.

---

## 1) صفحة تسجيل الدخول (`/login`)

| البند | الوصف |
|--------|--------|
| **الملف** | `frontend/src/views/auth/LoginView.vue` |
| **السلوك الحالي** | تسجيل الدخول الموحّد (بريد أو جوال **مع كلمة المرور**) كما كان — **لم يُزال ولم يُكسر**. |
| **إضافة واجهة** | رابط جديد بجانب «إنشاء حساب» يوجّه إلى **`/phone`** بنص الترجمة `login.linkPhoneOtp`. |
| **مفاتيح الترجمة** | `login.linkPhoneOtp` في `frontend/src/i18n/ar.ts`, `en.ts`, وملفات اللغات الأخرى (`ur`, `hi`, `tl`, `bn`). |

**للفريق عند النشر:** تأكد من بناء الواجهة بعد سحب الترجمات؛ لا حاجة لتغيير إعدادات nginx خاصة بـ `/login` ما دام SPA يخدم نفس الجذر.

---

## 2) صفحة التسجيل التقليدي (`/register`)

| البند | الوصف |
|--------|--------|
| **الملف** | `frontend/src/views/auth/RegisterView.vue` |
| **السلوك** | تسجيل شركة/مالك بالبريد وكلمة المرور وبيانات النموذج — **لم يُستبدل** بمسار الجوال. |
| **رابط إلى الجوال** | **غير مضاف** على صفحة التسجيل؛ مسار «جوال فقط + OTP» يبدأ من `/login` → `/phone` أو الدخول المباشر إلى `/phone`. |

إذا رغبتم لاحقاً بإضافة سطر «أو سجّل برقم الجوال» على `/register` يمكن تنفيذه كتحسين UX منفصل.

---

## 3) مسار الويب بالجوال + OTP (إضافي، بدون كلمة مرور)

| المسار | الغرض |
|--------|--------|
| `/phone` | إدخال رقم الجوال وطلب OTP |
| `/phone/verify` | إدخال الرمز (مع `?phone=` في الاستعلام) |
| `/phone/onboarding` | محور يجلب حالة التسجيل ويوجّه للخطوة التالية |
| `/phone/onboarding/type` | اختيار فرد / شركة |
| `/phone/onboarding/individual` | اسم الفرد |
| `/phone/onboarding/company` | اسم المنشأة + مسؤول الاتصال → انتظار مراجعة |
| `/phone/onboarding/pending-review` | شركة قيد المراجعة |
| `/phone/onboarding/done` | بعد إكمال الفرد — إرشاد وربط لاحق بشركة |

**الصفحات (Vue):** مجلد `frontend/src/views/phone/`.

**الحارس:** في `frontend/src/router/index.ts` — المستخدمون بدور `phone_onboarding` يُعاد توجيههم إلى مسار `/phone/...` إن حاولوا الدخول لباقي التطبيق؛ **يُستثنى** زيارة الصفحات ذات `meta.publicPage` (مثل `/landing`) لعرض تسويقي مع شريط عودة.

---

## 4) واجهات الـ API (Backend)

**قاعدة:** `POST /api/v1/auth/...` (حسب إعداد `apiClient` في الواجهة).

| الطريقة | المسار | ملاحظات |
|---------|--------|----------|
| POST | `/auth/phone/request-otp` | جسم: `phone` — رسالة عامة (لا تكشف وجود الرقم) |
| POST | `/auth/phone/verify-otp` | جسم: `phone`, `otp` — يعيد توكن + مستخدم + أعلام التدفق |
| GET | `/auth/registration-status` | يتطلب `Authorization: Bearer` |
| POST | `/auth/complete-account-type` | صلاحية `phone_registration.flow` |
| POST | `/auth/complete-individual-profile` | |
| POST | `/auth/complete-company-profile` | |

**لوحة المنصة (مراجعة شركات):** واجهة الويب ` /admin/registration-profiles ` + مسارات API المنصة كما في المشروع (للمشغّلين المصرّح لهم).

## 5) إعدادات النشر (Production)

| المتغير / الإعداد | الغرض |
|-------------------|--------|
| Twilio (أو ما يعادله) في `config/saas.php` / `.env` | إرسال SMS حقيقي |
| **عدم** تفعيل `PHONE_OTP_FAKE_PLAINTEXT` في الإنتاج | لا يُعاد OTP كنص في الاستجابة أو السجلات بشكل غير آمن |
| `PHONE_OTP_TTL`, الحدود الأخرى | ضبط حسب سياسة المنتج |

راجع تعليقات `backend/.env.example` تحت قسم تسجيل الجوال + OTP.

---

## 6) تحقق سريع قبل/بعد النشر (ويب)

1. `/login` — دخول بحساب قديم (بريد + كلمة مرور) يعمل.
2. من `/login` — الرابط «دخول برمز الجوال» يفتح `/phone`.
3. طلب OTP ثم التحقق (staging مع تسجيل/مزود تجريبي حسب البيئة).
4. إكمال فرد وصولاً إلى `/phone/onboarding/done` والروابط (تسجيل شركة / دخول شركة / هبوط).
5. مسار شركة حتى `pending-review` ووصول المشغّل لصفحة طابور المراجعة.

**اختبارات آلية (مرجع):**  
`backend/tests/Feature/Auth/PhoneRegistrationFlowTest.php` — مضمّن في `scripts/staging-gate.sh` و`Makefile` (`staging-gate`).

**E2E دخان واجهة:** `frontend/e2e/phone-onboarding-smoke.spec.ts`.
