# مركز التنبيهات والمتابعة

## 1) الهدف

تقديم طبقة موحّدة داخل إدارة المنصة لعرض ما **يحتاج انتباهك الآن** بشكل واضح وقليل الضوضاء، مع روابط مباشرة (Deep Link) إلى العنصر نفسه.

## 2) حدود المرحلة

- هذه المرحلة **قراءة وتنبيه فقط**.
- لا تنفيذ إصلاحي، لا إجراءات مالية، ولا أي remediation domain-sensitive.
- دعم: القسم الرئيسي في الصفحة، الجرس، وصفحة جميع التنبيهات.

## 3) أنواع التنبيهات

- اعتماد
- مالي
- دعم
- تشغيلي
- متابعة
- قرار
- حوكمة

## 4) مصادر التنبيهات

1. اعتماد منشأة (`registration_profiles` pending review)
2. اعتماد نموذج مالي (`companies.financial_model_status = pending_review`)
3. تذاكر الدعم ذات الأولوية (open/escalated/in_progress)
4. الحوادث التشغيلية العالية/الحرجة
5. المتابعات المجدولة القريبة (controlled actions)
6. القرارات التي تتطلب follow-up
7. طلبات الإلغاء المعلقة كعنصر تدخل/اعتماد

## 5) قاعدة Deep Link

كل تنبيه يحتوي:

- `target_route`
- `target_params`
- `target_type`
- `target_id`

وعند الضغط:

1. يعلّم كمقروء محليًا في الواجهة (بدون Workflow معقد في هذه المرحلة)
2. ينقلك مباشرة للهدف (وليس صفحة عامة فقط)
3. عند غياب الصلاحية يظهر: **ليس لديك صلاحية للوصول**

## 6) الصلاحيات

- تمت إضافة مفتاح: `platform.notifications.read`.
- المخرجات role-aware في الخادم: كل مصدر يُضمَّن فقط إذا كانت صلاحية المصدر نفسه متاحة.
- منع تسريب cross-role عبر تجميع مشروط لكل مصدر.

## 7) الترتيب وتقليل الضوضاء

- الأولوية: `critical` ثم `high` ثم `medium` ثم `informational`.
- grouping على مصادر عالية الضوضاء (مثل الدعم/الاعتمادات) بدل تكرار عناصر متطابقة.
- قسم الصفحة الرئيسية يعرض 4-8 عناصر فقط (Top attention).

## 8) المسمّيات العربية المعتمدة

- مركز التنبيهات والمتابعة
- يحتاج انتباهك الآن
- جميع التنبيهات
- غير المقروءة
- تحتاج إجراء
- اعتماد
- دعم
- تشغيل
- مالي
- متابعة
- عرض الكل
- فتح
- مراجعة
- اعتماد
- فتح التذكرة
- عرض الحادث
- ليس لديك صلاحية للوصول

## 9) معايير الإغلاق

- يوجد مركز تنبيهات موحد في Backend + Frontend.
- يوجد قسم “يحتاج انتباهك الآن” في الصفحة الرئيسية.
- الجرس يعمل مع عدّاد غير المقروء.
- صفحة “جميع التنبيهات” تعمل مع فلاتر أساسية.
- كل تنبيه يملك target صالح وروابط مباشرة.
- الصلاحيات محترمة، وعند عدم الصلاحية يظهر سلوك آمن واضح.
- لا وجود لأي تنفيذ remediation أو إجراء مالي حساس ضمن هذه الطبقة.
