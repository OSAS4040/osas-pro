# تقرير تقدم المرحلة 3 — أمان وعقود الأخطاء

**حالة الوثيقة:** **تقدم** — عقود الأخطاء والتعارض تتطلب مواءمة مستمرة مع واجهات العميل.

## 1. الهدف

تثبيت شكل استجابات التعارض (409) والرموز المتوقعة حتى لا تتكسر الواجهات الصامتة أو معالجات إعادة المحاولة.

## 2. قائمة تحقق

| البند | الملاحظات |
|--------|------------|
| عقد خطأ التعارض | `ConflictErrorContractTest` |

## 3. الاختبارات

- مجموعة PHPUnit: **`phase3`**
- الأمر: `composer test:phase3`
- **واجهة (Vitest):** `cd frontend && npm run test:phase3` — `apiErrorCodes` و`loginApiErrors`.

## 4. فجوات

- توسيع المصفوفة عند إضافة رموز تعارض جديدة لمجالات أخرى (مخزون، فواتير، إلخ) مع توثيق في مرجع API.
