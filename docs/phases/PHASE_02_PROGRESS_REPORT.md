# تقرير تقدم المرحلة 2 — محفظة واجتماعات وتدفقات تشغيلية

**حالة الوثيقة:** **تقدم** — التدفقات المالية والتشغيلية تتوسع باستمرار.

## 1. الهدف

حماية مسارات حرجة مثل طلب شحن المحفظة وسير عمل الاجتماعات (دفعة 2) من الانحدار أثناء تغييرات الـ API أو الصلاحيات.

## 2. قائمة تحقق

| البند | الملاحظات |
|--------|------------|
| سير عمل طلب شحن المحفظة | `WalletTopUpRequestWorkflowTest` |
| اختبارات اجتماعات الدفعة 2 | `MeetingsMvpBatch2Test` |

## 3. الاختبارات

- مجموعة PHPUnit: **`phase2`**
- الأمر: `composer test:phase2`
- **واجهة (Vitest):** `cd frontend && npm run test:phase2` — دفعات/عمليات/سياق تنقل (جزء من التدفق التشغيلي بجانب المحفظة/الاجتماعات في الخلفية).

## 4. فجوات

- مراجعة تغطية حالات الرفض/الحدود لطلبات المحفظة عند إضافة قواعد عمل جديدة.
- Vitest مخصص لمسارات **الاجتماعات** في الواجهة عند توفر وحدات اختبار.
