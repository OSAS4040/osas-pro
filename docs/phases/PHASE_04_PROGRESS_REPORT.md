# تقرير تقدم المرحلة 4 — محاسبة ودفتر قيود

**حالة الوثيقة:** **تقدم** — القيود والترحيلات حساسة للتغييرات في نموذج البيانات.

## 1. الهدف

ضمان أن التراجع عن الترحيل أو سيناريوهات دفتر القيود الحرجة لا تكسر التوازن أو العلاقات المتوقعة.

## 2. قائمة تحقق

| البند | الملاحظات |
|--------|------------|
| ترحيل وتراجع القيود | `LedgerPostingRollbackTest` |

## 3. الاختبارات

- مجموعة PHPUnit: **`phase4`**
- الأمر: `composer test:phase4`
- **واجهة (Vitest):** `cd frontend && npm run test:phase4` — عرض فواتير، وقت محلي، تسميات حقول.

## 4. فجوات

- إضافة اختبارات تكامل للفواتير ↔ القيود عند تغيير سياسات الإقفال المحاسبي.
