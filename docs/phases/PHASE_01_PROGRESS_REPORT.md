# تقرير تقدم المرحلة 1 — SaaS وكتالوج الباقات

**حالة الوثيقة:** **تقدم** — لا تُغلق نهائياً ما دام كتالوج الباقات والاشتراكات يتطور (إضافات باقات، حدود، امتدادات منصة).

## 1. الهدف

ضمان أن بوابات SaaS الأساسية (عرض/تحديث كتالوج الباقات، تطابق الخطة مع الاشتراك حيث ينطبق) تبقى قابلة للتحقق آلياً دون كسر عقود الـ API.

## 2. قائمة تحقق

| البند | الملاحظات |
|--------|------------|
| اختبارات كتالوج الباقات تمر في CI | `SaasPlanCatalogGateTest` |
| أي تغيير في `Plan`/`Subscription` يُراجع مع اختبارات SaaS | يدوي + مراجعة فرق |
| توثيق نقاط النهاية الحساسة للمنصة | يرتبط بمرحلة 6 عند التقاطع |

## 3. الاختبارات

- مجموعة PHPUnit: **`phase1`**
- الأمر: `composer test:phase1`
- **ملفات ضمن المجموعة:** `SaasPlanCatalogGateTest`, `PlanAddonSubscriptionTest`, `SaasControllerContainerTest`, ووحدة `tests/Unit/Support/SaasPlatformAccessTest.php` (قائمة السماح لمشغّل المنصة).
- **واجهة (Vitest):** `cd frontend && npm run test:phase1` — تنقل إدارة المنصة وتلميحات ترحيبية (طبقة إدارية بجانب كتالوج الباقات في الخلفية).

## 4. فجوات

- إضافة اختبارات Vitest مباشرة لشاشات **باقات المستأجر** عندما تُستقر واجهاتها.
