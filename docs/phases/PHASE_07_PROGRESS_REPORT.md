# تقرير تقدم المرحلة 7 — مسار تشغيلي كامل (ما قبل الإنتاج)

**حالة الوثيقة:** **تقدم** — المسار الشامل يُحدَّث مع كل تغيير في نموذج الفاتورة أو المخزون أو المحفظة.

## 1. الهدف

محاكاة عميل HTTP عبر سلسلة طويلة: عميل، مركبة، أمر عمل، فاتورة، محفظة، مع فحوصات `trace_id`، عدم تكرار الفاتورة، مخزون غير سالب، وميزان القيود.

## 2. قائمة تحقق

| البند | الملاحظات |
|--------|------------|
| المسار الكامل | `RealWorkflowApiTest` |
| مجموعات إضافية للجاهزية | `pre-production`, `production-readiness` |

## 3. الاختبارات

- مجموعة PHPUnit: **`phase7`**
- الأمر: `composer test:phase7`
- **واجهة (Playwright):** `cd frontend && npm run test:phase7` — `readiness-gate-public`, `auth-public`, ثم `production-real-flow` (يتطلب الأخير `PW_LOGIN_EMAIL` و`PW_LOGIN_PASSWORD` وإلا يُتخطّى). سلسلة Vitest ثم هذه المجموعة: `npm run test:phases:fe:with-e2e`.

## 4. فجوات

- زمن التشغيل أطول من المراحل الأخرى؛ يُفضَّل تشغيله في CI ليلاً أو كمجموعة منفصلة مع مهلة كافية.
- عند إضافة خطوات جديدة (مثل ZATCA أو أصول ثابتة)، يُوسَّع الاختبار أو يُضاف مسار موازٍ موثَّق.
