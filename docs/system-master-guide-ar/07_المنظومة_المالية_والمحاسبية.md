# المنظومة المالية والمحاسبية

## الدورة المالية
إنشاء التزام مالي -> تنفيذ معاملة -> تسجيل أثر -> مطابقة -> معالجة استثناءات.

## الدورة المحاسبية
قيد/ترحيل -> ميزان مراجعة -> تقارير -> عكس عند الحاجة ضمن ضوابط.

## نموذج الفوترة التعاقدية التلقائي (معتمد)
- لا تُرسل فاتورة مزود الخدمة إلى العميل.
- عند اكتمال الخدمة وتوثيق أمر العمل، يقوم النظام تلقائيًا بإصدار:
  - فاتورة مبيعات للعميل وفق سعر التعاقد مع العميل.
  - فاتورة مشتريات لمزود الخدمة وفق سعر التعاقد مع المزود.
- المستندان يرتبطان بنفس أمر العمل ودورة التسوية.
- الفوترة هنا تلقائية بدون مسار اعتماد يدوي لفاتورة المزود.

## قواعد محاسبية تشغيلية
- فصل سعر البيع (عميل) عن سعر الشراء (مزود) إلزامي.
- الفارق يمثل هامش المنصة ويُحسب من عقود التسعير المعتمدة.
- يمنع كشف سعر شراء المزود للعميل، ويمنع كشف سعر بيع العميل للمزود.

## ضوابط أساسية
- financial.protection للمسارات الحساسة.
- idempotent لعمليات الدفع/التحويل/البيع الحساسة.
- reconciliation مجدول لاكتشاف الانحرافات.
- منع التكرار المحاسبي لنفس أمر العمل (idempotency key).
- عند غياب تسعير تعاقدي صالح: تحويل الحالة إلى استثناء تسعير بدل إصدار خاطئ.
