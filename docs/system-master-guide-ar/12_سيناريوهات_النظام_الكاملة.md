# سيناريوهات النظام الكاملة

## يومية
إدارة عميل/مركبة/أمر عمل/فاتورة/تحصيل.

## سيناريو تعاقدي موحد (العميل + مزود الخدمة + إدارة المنصة)
1. العميل يشحن رصيد مركبة أو ينشئ أمر عمل للمركبة.
2. العميل لا يصدر فواتير ولا عروض أسعار، ويطّلع فقط على الفواتير الصادرة عليه.
3. مزود الخدمة المعتمد يبحث باللوحة أو رقم أمر العمل.
4. النظام يعرض تفاصيل أمر العمل وحالة الرصيد قبل التنفيذ.
5. المزود ينفذ الخدمة ويوثقها داخل النظام.
6. النظام ينشئ تلقائيًا مستندين ماليين من نفس أمر العمل:
   - فاتورة مبيعات للعميل حسب تعاقد العميل.
   - فاتورة مشتريات للمزود حسب تعاقد المزود.
7. إدارة المنصة تتابع النتائج والتقارير والاستثناءات، دون اعتماد يدوي لفاتورة المزود في المسار القياسي.

## إدارية
تحديث صلاحيات، تفعيل ميزات، ضبط تكاملات.

## مالية ومحاسبية
سداد، تحويل، عكس، مطابقة.

## ذكية
تحول إشارة إلى حادثة ثم معالجتها حتى الإغلاق.

## استثنائية
فشل صلاحية، feature inactive، فشل توافق مالي، تعارض lifecycle.
