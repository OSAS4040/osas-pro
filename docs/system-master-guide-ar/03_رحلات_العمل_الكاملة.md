# رحلات العمل الكاملة

## من التسجيل حتى الإغلاق
1. دخول/تسجيل/OTP.
2. تحديد سياق الحساب والبوابة.
3. دخول المستخدم لبوابته المخصصة.
4. تنفيذ العمليات (تشغيلية/مالية/إدارية).
5. تحديث الحالات وإرسال الإشعارات.
6. معالجة الاستثناءات (تصعيد/إغلاق incident) عند الحاجة.
7. إغلاق الحالة مع أثر مالي/تشغيلي/إداري.

## الرحلة التشغيلية المعتمدة (عميل + مزود خدمة + إدارة منصة)
1. العميل يشحن رصيد المركبة أو ينشئ أمر عمل مرتبطًا بالمركبة.
2. عند وصول المركبة إلى مزود خدمة معتمد ومسجل بالمنصة، يبحث المزود برقم أمر العمل أو رقم اللوحة.
3. النظام يعرض للمزود بيانات أمر العمل وحالة الرصيد (متوفر/غير متوفر).
4. المزود ينفذ الخدمة ويوثق التنفيذ (بنود/وقت/ملاحظات/مرفقات).
5. المزود يصدر فاتورة التنفيذ داخل النظام.
6. فاتورة المزود لا تُرسل للعميل إطلاقًا، وتبقى ضمن سياق المنصة.
7. النظام يصدر تلقائيًا مستندين حسب التعاقدات:
   - فاتورة مبيعات للعميل بسعر التعاقد مع العميل.
   - فاتورة مشتريات لمزود الخدمة بسعر التعاقد مع المزود.
8. ترحيل الأثر المالي وإغلاق أمر العمل وتحديث التقارير.

## رحلات أساسية
- رحلة الخدمة: أمر عمل -> تنفيذ موثق -> إصدار تلقائي (مبيعات عميل + مشتريات مزود) -> ترحيل -> إغلاق.
- رحلة Platform: متابعة شركات -> ضبط اشتراكات وتعاقدات -> متابعة incidents والاستثناءات.
- رحلة Support: تذكرة -> ردود -> SLA -> إغلاق.
- رحلة Intelligence: signal -> candidate -> incident -> decision -> action -> close.
