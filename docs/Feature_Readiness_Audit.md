# Feature readiness audit — تجربة المستخدم والربط مع الـ API

**تاريخ المراجعة:** 2026-04-05  
**نطاق:** مستودع `new-project-3` (Laravel API + Vue Staff/Fleet/Customer).

## 1) منهجية المراجعة

| المصدر | ماذا يغطي | القيود |
|--------|-----------|--------|
| **اختبارات خلفية** | `docker compose exec app php artisan test` — **299 اختباراً ناجحاً** (عقود API، صلاحيات، انتقالات حالة، محفظة، فواتير، مطابقة مالية، إلخ) | لا تغطي كل شاشة واجهة يدوياً. |
| **فحص مسارات API** | `backend/routes/api.php` مقابل `frontend/src/router/index.ts` ونداءات `apiClient` | اختلاف تسمية المسار يظهر فقط عند تصفح يدوي أو e2e. |
| **فحص شفرة الواجهة** | `grep` لعناصر «قريباً»، تخزين محلي فقط، ورسائل `alert` المضللة | لا يعوّض اختبار قابلية استخدام حقيقي مع مستخدمين. |

**تعريف الحالات:**

- **Ready:** مسار واجهة + نقاط API مكافئة (أو تدفق واضح بدون API كـ طباعة المتصفح) تعمل لمن لديه الصلاحية؛ لا يوجد في الكود ما يدل على «واجهة فقط» بدون خادم لهذه الوظيفة الأساسية.
- **Partial:** يعمل الجزء الأساسي أو الـ API موجود لكن الواجهة ناقصة، أو **بيانات محلية فقط**، أو **يعتمد على أعلام بناء**، أو رسالة المستخدم **مضللة** سابقاً تم تخفيفها حيث وُجدت.
- **Not implemented:** شاشة توضيحية بدون ربط API للسيناريو المعلن، أو زر صريح «قريباً»، أو إعادة توجيه دائمة تلغي الميزة.

## 2) ملخص تنفيذي

- **النواة المالية والتشغيلية والمحفظة والفواتير وأوامر العمل** مؤكَّدة بشكل قوي عبر مجموعة الاختبارات.
- **لم تُنفَّذ مراجعة يدوية كاملة** لكل زر في كل دور؛ المصفوفة أدناه تقوم على الكود ونتيجة الاختبارات.
- **تصدير PDF (فاتورة + جواز مركبة):** رسالة خطأ موحّدة للمستخدم (`PDF_EXPORT_FAIL_AR` في `frontend/src/constants/pdfExportMessages.ts`)؛ زر مع **تحميل وتعطيل أثناء التصدير**؛ **نجاح** عبر `toast.success`؛ **لا فشل صامت** (إزالة `catch { window.print() }` من `InvoiceShowView.vue` دون إشعار؛ تنظيف عقدة الاستنساخ في `finally` عند استخدام `html2canvas`).
- **إخفاء/تعطيل واجهات غير تشغيلية (2026-04-05):** مسارات بلا سلوك منتج نهائي أُزيلت من التنقل والبحث السريع ولوحة الأوامر؛ تُعاد توجيهها إلى `access-denied` بسبب `inactive` أو `preview` مع نص مهني (`meta.featureInactive`، `meta.unavailablePreview`). راجع `docs/UAT_FEATURE_MATRIX.md` §2.
- **توحيد تفعيل الميزات في الواجهة (2026-04-05):** منطق واحد في `frontend/src/config/staffFeatureGate.ts` يغذي الراوتر (`beforeEach` + `ensureBusinessProfileLoaded`)، القائمة، البحث، الـ Command Palette، لوحة التحكم، وشريط أدوات التقارير. لا إعادة توجيه صامتة عند المنع: `access-denied` مع `reason` (`feature`، `permission`، إلخ). تفاصيل الجدول في `docs/UAT_FEATURE_MATRIX.md` §3.
- **توحيد أخطاء الـ API والتحميل (2026-04-05):** `frontend/src/lib/apiClient.ts` يغطي 403/404/409/422/402/423/429/503/5xx والشبكة والمهلة عبر Toast؛ الرسائل الكبيرة/التقنية تُصفّى؛ الطلبات التي تعرض خطأ في الشاشة تستخدم `skipGlobalErrorToast` مع `summarizeAxiosError` في `frontend/src/utils/apiErrorSummary.ts`. الوثيقة المرجعية: `docs/Frontend_Error_Handling_Standard.md`؛ قائمة UAT سريعة: `docs/UAT_FEATURE_MATRIX.md` §4.

## 4) فجوات تجربة مستخدم موصى بمعالجتها (Backlog)

1. **توحيد الرسائل:** استبدال `alert()` المتبقي في صفحات حرجة بـ `useToast` حيثما أمكن (أسطول، مخزون، إدارة).  
2. **مستندات المنشأة:** القائمة الوصفية على الخادم؛ لاستكمال **PARTIAL → Ready** يلزم مسار رفع ملفات حقيقي (تخزين مؤسسي) إن رُغب بمرفقات متزامنة.  
3. ~~**صفحات HR التوضيحية:**~~ التوقيع/WPS/مكتبة العمل ما زالت غير مفعّلة كما في الجدول؛ **الأرشفة الإلكترونية** أصبحت شاشة تشغيلية أولية مربوطة بـ `documents_registry` (راجع صف الجدول أعلاه).  
4. **E2E:** توسيع `frontend/e2e` ليشمل تدفقاً واحداً على الأقل للفاتورة + أمر عمل (اليوم: auth + landing فقط).  
5. **تحديث هذه الوثيقة:** مع كل PR يمس واجهة رئيسية — صف واحد في المصفوفة أو قسم «Partial».

## 5) فحص يدوي مقترح — تصدير PDF (فاتورة / جواز مركبة)

| السيناريو | التوقع |
|-----------|--------|
| فاتورة عادية بدون شعارات خارجية | زر PDF → «جاري التصدير…» → تنزيل + toast نجاح. |
| فاتورة بشعار/QR (صور من نفس المصدر) | كما أعلاه؛ إن منع المتصفح التقاط الصور (`CORS`/تلوين) → toast خطأ بنص `PDF_EXPORT_FAIL_AR` وليس فتح طباعة مفاجئ. |
| فاتورة طويلة (عدة صفحات) | عدة صفحات في PDF دون تعليق واجهة الدون إعادة ضغط الزر (التعطيل أثناء التشغيل). |
| جواز مركبة بعد تحميل البيانات | تصدير ملخص ناجح؛ عند فشل تحميل المكتبة تظهر رسالة خطأ موحّدة. |
| فشل محاكى (قطع الشبكة قبل `import('jspdf')`) | خطأ ظاهر للمستخدم، لا `catch` فارغ. |
