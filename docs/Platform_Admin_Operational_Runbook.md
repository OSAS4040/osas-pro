# الدليل التشغيلي بعد النشر — إدارة المنصة

## ما الذي يراقبه فريق التشغيل

- صحة API: `/api/v1/platform/*` (استجابة/زمن).
- أخطاء 4xx/5xx لمسارات إدارة المنصة.
- Logs تطبيق Laravel (استثناءات صلاحيات، أخطاء DB).
- سلوك الواجهة في `/platform/*` (فشل تحميل/نوافذ رفض).

## مؤشرات الاستقرار السريع

- تسجيل دخول مشغّل المنصة ناجح.
- `/platform/overview` تحمل خلال زمن مقبول.
- Incident Center وCommand Surface وNotifications تعمل.
- لا ارتفاع غير طبيعي في أخطاء `403/500`.

## كيف نعرف أن هناك مشكلة

- تكرار `403` لأدوار مصرح لها.
- `500` متكرر في APIs المنصة.
- فشل فتح صفحات حرجة أو فشل Deep Links.
- فشل smoke checklist.

## أول نقطة فحص عند ظهور خلل

1. تحقق من صحة الإصدار المنشور وبيئة الـ env.
2. راجع نتيجة `migrate:status`.
3. تحقق من صلاحيات الدور المعني.
4. راجع logs آخر 15 دقيقة.

## متى يتم التصعيد

- عند فشل مسار حرج (overview / incidents / command / notifications).
- عند أي تسريب صلاحيات.
- عند عدم استقرار متكرر بعد إعادة محاولة قصيرة.

## إلى من يتم التصعيد

- On-call Backend
- On-call Frontend
- Release Owner / Platform Owner

## إجراء تشغيلي فوري مقترح

- Freeze نشر إضافي.
- تشغيل smoke سريع.
- قرار: Hotfix محدود أو Rollback حسب `GO/NO-GO`.
