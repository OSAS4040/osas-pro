# تقرير التنفيذ الآمن — مسرد المنصة، البيئة، والبوابات (v1)

**التاريخ (مرجع):** نيسان 2026  
**النطاق:** ما يُنفَّذ على الأرض دون إعادة بناء نموذج اشتراك «عميل نهائي ↔ منصة»، وبحد أدنى من المخاطر التشغيلية.

**سياسة التشغيل الملزمة:** [`Staging_Governance_Policy.md`](./Staging_Governance_Policy.md) · [`Staging_Gate_Mandatory_Policy.md`](./Staging_Gate_Mandatory_Policy.md) · [`Branch_Protection_And_Review_Policy.md`](./Branch_Protection_And_Review_Policy.md).

**ترتيب التنفيذ:** [`Execution_Order_Asas_Pro.md`](./Execution_Order_Asas_Pro.md).

## 1. ما تم تنفيذه

### 1.1 مسرد واجهة ثابت

- صفحة **`/about/taxonomy`** (مسرد الكيانات والبوابات): تعريف **منصة أسس برو**، **الشركة المشتركة (المستأجر)**، **العميل النهائي**، وعلاقة ذلك بالبوابات (Staff / Fleet / Customer / لوحة المنصة).
- روابط من **لوحة الأدمن** (`/admin`)، **بيانات النشر** (`/about/deployment`)، والشريط الجانبي (نسخة الدعم)، لتسهيل الوصول للفريق دون توثيق خارجي إلزامي.

### 1.2 تفعيل بوابات اختيارية (واجهة فقط — وقت البناء)

- المتغير **`VITE_ENABLED_PORTALS`**: قائمة مفصولة بفواصل من المعرفات `fleet`, `customer`, `admin`.
- **الافتراضي الآمن:** غير مضبوط أو فارغ ⇒ **تفعيل كل البوابات الاختيارية** (لا تغيير سلوك النشر الحالي).
- عند التحديد الصريح: تُفعَّل **فقط** البوابات المذكورة (قائمة بيضاء).
- **بوابة فريق العمل (Staff)** لا تُعطَّل من هذا المفتاح — مسار التشغيل الأساسي للورشة يبقى متاحاً.

**أثر المنتج:** إخفاء عناصر القائمة، البحث السريع، دليل المستخدم الذكي، ومبدّل البوابة، مع **حراسة Router** لمسارات `/fleet-portal`، `/customer`، و`/admin` عند التعطيل.

### 1.3 الخادم (موجود مسبقاً — مُثبَّت بالاختبارات)

- **`SAAS_PLATFORM_ADMIN_EMAILS`**: قائمة بريد لمشغّلي المنصة (قراءة المشتركين، تعديل كتالوج الباقات عند إغلاق تعديل المستأجرين للجدول العالمي).
- **`SAAS_ALLOW_TENANT_PLAN_CATALOG_EDIT`**: يُفضَّل `false` في إنتاج متعدد المستأجرين.
- اختبارات Feature لبوابات الخطة و`GET /api/v1/admin/companies`؛ واختبارات Unit لـ **`SaasPlatformAccess`**.

## 2. الأخطاء التي تُعالَج بهذا النطاق

| المشكلة | المعالجة |
|--------|-----------|
| خلط «عميل المنصة» مع «عميل الورشة» | مسرد لغوي واضح + تسمية في الوثائق الداخلية |
| ظهور بوابات غير مباعة في v1 | `VITE_ENABLED_PORTALS` + حراسة مسارات |
| ضعف وضوح صلاحية لوحة المنصة | تذكير بربط البريد بالخادم في لوحة الأدمن + API 403 |

## 3. مقترحات تقنية لاحقة (غير منفّذة هنا — عمداً)

1. **هوية منصة مستقلة:** مستخدمون/أدوار `platform_*` منفصلة عن مالك المستأجر، مع مسارات `/platform/...` وتدقيق (Audit) موحّد.
2. **اشتراك عميل نهائي ↔ المنصة:** يتطلب جداول وفوترة وامتثالاً — يُؤجَّل حتى قرار منتج وصياغة عقد.
3. **دفتر إيرادات منصة:** لا يُستنتج من تقديرات الواجهة؛ يحتاج مصدر حقيقة محاسبي وربط دفع.
4. **تعطيل بوابات من الخادم:** إن لزم فرض سياسة مركزية، يمكن لاحقاً لاحقاً إرجاع `GET /api/v1/bootstrap` مع أعلام بوابات (يتطلب تنسيقاً مع الخلفية).

## 5. ملخص

هذا الإصدار يطبق **المرحلة الأكثر أماناً**: وضوح المفاهيم، تحكماً اختيارياً في ظهور البوابات، واختبارات تغطي بوابات SaaS على الخادم — دون ترحيل بيانات مالية أو اشتراك جديد للعميل النهائي مع المنصة.
